The happydesigns voice is thoughtful, warm, precise, and practical. It should sound confident but not loud, technical but still readable, and clear before clever.

Good writing helps people understand a decision, take an action, or trust the system.

## Voice Attributes

| Attribute | Meaning |
| --- | --- |
| Thoughtful | Explain the useful reason behind a decision. |
| Warm | Sound human and approachable without becoming casual filler. |
| Precise | Use concrete nouns, direct verbs, and specific outcomes. |
| Practical | Help people understand, decide, or act. |
| Maintainable | Prefer rules and patterns that can keep working over time. |

## Writing Principles

- Say what the user can do or understand next.
- Use concrete nouns and direct verbs.
- Keep claims grounded in real value.
- Explain decisions through durable rules.
- Avoid hype, vague promises, and decorative wording.
- Let technical writing stay readable.

## Marketing Copy

Marketing copy should be warm and benefit-oriented, but not generic. It should connect design quality to durable web work.

Before:

> Supercharge your brand with next-level digital experiences.

After:

> Build a clearer web presence with a system that can grow.

Before:

> We create stunning solutions for every business.

After:

> We design websites and web systems that stay clear, useful, and maintainable.

## Documentation Copy

Documentation copy should help people apply the brand consistently. It can be technical, but it should remain readable.

Use documentation copy to:

- Define roles and boundaries.
- Explain when to use a pattern.
- Name exceptions clearly.
- Connect implementation details to brand behavior.

Good:

> Use coral for focus, activation, and signature details. Do not use it as a general background color.

Too vague:

> Use coral to make things feel more branded.

## Interface Copy

Interface copy should help users act. It should not advertise the interface back to them.

Good labels:

- `Save changes`
- `Copy token`
- `Start a project`
- `Review details`
- `Upload logo`

Avoid labels that are too generic when the action is known:

- `Submit`
- `Continue`
- `Click here`
- `Learn more`

`Continue` and `Learn more` can still work when they are genuinely the clearest label in context.

## CTA Copy

Calls to action should be clear and grounded. Prefer direct verbs over hype.

| Context | Better CTA |
| --- | --- |
| Contact section | `Start a project` |
| Documentation | `View logo rules` |
| Technical utility | `Copy token` |
| Form workflow | `Save changes` |
| Review flow | `Review details` |

## Error And Validation Copy

Errors should be calm, specific, and recovery-oriented. Avoid blame and avoid dead-end messages.

Before:

> Something went wrong.

After:

> The file could not be uploaded. Check the format and try again.

Before:

> Invalid input.

After:

> Enter a valid email address.

Before:

> Failed.

After:

> The preview could not be generated. Try again in a moment.

## Tone By Context

| Context | Tone |
| --- | --- |
| Landing page | Warm, confident, concise. |
| Documentation | Precise, practical, rule-based. |
| Product UI | Short, action-oriented, calm. |
| Technical examples | Clear, exact, readable. |
| Errors | Specific, calm, recovery-oriented. |

## Do

- Use concrete nouns and direct verbs.
- Keep interface labels short and action-oriented.
- Make technical text readable.
- Explain the reason behind important rules.
- Keep claims grounded and specific.

## Do Not

- Do not rely on hype or exaggerated promises.
- Do not over-explain conventional controls.
- Do not use clever wording when clarity matters.
- Do not let brand copy obscure product behavior.
- Do not make errors sound dramatic or vague.
