Logo documentation keeps source files, output files, clearspace expectations, and common placements in one place.

## Assets

| Asset | Path | Usage |
| --- | --- | --- |
| Wordmark | `app/assets/logos/happydesigns.svg` | Default identity mark for headers, covers, and documentation. |
| Dark wordmark | `app/assets/logos/happydesigns-dark.svg` | Use on graphite or other dark surfaces. |
| Favicon SVG | `app/assets/logos/happydesigns-favicon.svg` | Browser tab and small vector contexts. |
| App icon PNG | `/app-icon.png` | Touch icon and square app contexts. |

## Usage

- Use the wordmark where the brand name needs to be readable.
- Use the app icon only when the surrounding product context already supplies the name.
- Keep clearspace around the logo; do not place it tight against text, borders, or controls.
- Use the dark wordmark on graphite surfaces.

## Placement

- Header: wordmark.
- Browser tab: favicon.
- App surface: square icon.
- Documentation cover or brand overview: wordmark with supporting brand field.
