Docs surfaces should make decisions easy to read, cite, and reuse. Prose components carry explanation, examples, and implementation detail without becoming decorative.

## Callouts For Decisions

::id-nuxt-ui-docs-link{links="Callout|https://ui.nuxt.com/docs/typography/callout"}
::

Use notes when a decision needs extra context, a warning, or a concrete implementation rule.

::code-preview
:::callout{title="Use callouts for decisions" icon="i-lucide-info"}
Callouts should clarify a decision, warning, or implementation note. They should not be used as decoration between normal paragraphs.
:::

## File Trees

::id-nuxt-ui-docs-link{links="CodeTree|https://ui.nuxt.com/docs/typography/code-tree"}
::

Use code trees when one implementation depends on several files. Keep the tree focused on the minimum structure needed to understand the setup.

::code-preview
:::code-tree{default-value="nuxt.config.ts" expand-all}
```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['@happydesigns/brand']
})
```

```ts [app/app.config.ts]
export default defineAppConfig({
  id: {
    defaultTheme: 'happydesigns'
  }
})
```

```css [app/assets/css/main.css]
@import "tailwindcss";
@import "@nuxt/ui";
```
:::

## Compact Inline Signals

::id-nuxt-ui-docs-link{links="Icon|https://ui.nuxt.com/docs/typography/icon; Badge|https://ui.nuxt.com/docs/typography/badge; Kbd|https://ui.nuxt.com/docs/typography/kbd"}
::

Use icons, badges, and keyboard hints only when they improve scanning or identify an exact action.

::code-preview
- :icon{name="i-lucide-sparkles"} marks an inline signal when the icon helps recognition.
- :badge[Stable] marks a compact state or version.
- :kbd{value="Ctrl"} :kbd{value="K"} opens search.
