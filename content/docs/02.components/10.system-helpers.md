System helpers should stay quiet and precise. They support the product shell, accessibility, and recovery paths; they should not become decorative controls.

## System Helper Rules

- Keep theme controls predictable: button in compact headers, switch or select in settings, scoped theme only in previews.
- Use mode-aware assets only when contrast or legibility changes between light and dark surfaces.
- Keep locale controls near content that changes language or formatting.
- Make auth forms direct, with one primary path and clear recovery states.
- Use error pages to explain what happened and where the user can go next.
