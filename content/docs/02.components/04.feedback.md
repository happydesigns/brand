Feedback should be calm, specific, and tied to the next useful action. Color should clarify state, not decorate the message.

## Feedback Patterns

::id-nuxt-ui-docs-link{links="UBanner|https://ui.nuxt.com/docs/components/banner; UAlert|https://ui.nuxt.com/docs/components/alert; UProgress|https://ui.nuxt.com/docs/components/progress; USkeleton|https://ui.nuxt.com/docs/components/skeleton; UToast|https://ui.nuxt.com/docs/components/toast; UEmpty|https://ui.nuxt.com/docs/components/empty"}
::

Use banners for broad notices, alerts for nearby state, progress and skeletons for waiting, toasts for finished actions, and empty states for a clear next step. Each pattern should make the state and the recovery path easier to understand.

## Feedback Copy

| State | Example |
| --- | --- |
| Success | `Changes saved.` |
| Progress | `Generating preview.` |
| Empty | `No projects yet. Start with the first project brief.` |
| Warning | `Check contrast before publishing.` |
| Error | `The token name is already used. Choose a unique name.` |

## Feedback Rules

- Match the component to the scope of the message: banner for global context, alert for nearby state, toast for completed actions.
- Pair every warning or error with the next useful action.
- Keep success messages short and avoid celebrating routine work.
- Use progress and skeleton states when waiting changes the layout or delays the next decision.
- Let semantic color identify state, then let the copy explain what happened.
