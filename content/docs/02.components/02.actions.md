Actions should be clear before they feel branded. A user should understand the main action, alternate paths, and supporting metadata without reading around the interface.

## Action Hierarchy

::id-nuxt-ui-docs-link{links="UButton|https://ui.nuxt.com/docs/components/button; UTooltip|https://ui.nuxt.com/docs/components/tooltip"}
::

Use one filled action for the strongest decision in a view. Secondary actions should stay visible without competing with it. Tooltips clarify compact controls, but should not hide required instructions.

## Action Rules

- Use graphite-filled primary buttons for strong actions.
- Use outlined or subtle secondary buttons for alternate paths.
- Keep coral available for focus, active state, and signature emphasis.
- Use icon-only buttons only when the symbol is familiar or has a tooltip.
- Avoid huge coral CTA surfaces by default.

## Action Copy

Write action labels as clear outcomes: `Save changes`, `Copy token`, `Open docs`, or `Start a project`.

Avoid vague labels such as `Submit`, `Click here`, or `Learn more` when a more specific action exists.
