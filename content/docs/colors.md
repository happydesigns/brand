Color decisions are documented as named tokens first, then mapped to interface intent. This keeps future brand guides portable while preserving local expression.

## Palette

| Token | Hex | Role |
| --- | --- | --- |
| `warmWhite` | `#FAF7F2` | Page background |
| `graphite` | `#242423` | Text and premium fills |
| `coral` | `#F28564` | Signature accent |
| `peach` | `#FFE4D4` | Logo field and atmosphere |
| `petrol` | `#2F6374` | Technical counterweight |
| `butter` | `#E8CE69` | Soft warning |
| `plum` | `#71617F` | Quiet information |
| `seafoam` | `#1FA573` | Constructive success |
| `sand` | `#E9E4DC` | Lines and muted surfaces |
| `rose` | `#BD4D49` | Restrained errors |

## Semantic Roles

- `primary`: coral for focus and brand emphasis.
- `secondary`: petrol for analytical or technical actions.
- `success`: seafoam for constructive confirmations.
- `info`: plum for quiet metadata and informational states.
- `warning`: butter for caution without alarm.
- `error`: rose for destructive or blocking states.
- `neutral`: sand for borders, panels, and muted controls.

## Usage Rules

- Use warm white and graphite as the stable contrast pair.
- Use coral sparingly so it keeps its value as the signature accent.
- Use sand for visible structure: borders, separators, muted panels, and grids.
- Avoid filling large surfaces with saturated accent colors.
