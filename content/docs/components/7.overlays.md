Overlays should support the current task without making the interface feel heavier than it needs to be.

::component-family-showcase{family="overlays"}
::

## Rules

- Use modals for focused decisions.
- Use slideovers for supporting work that still belongs to the current page.
- Use popovers for compact contextual detail.
- Use tooltips for short clarifications, not required instructions.
- Keep destructive choices explicit and recoverable.

## Covered Components

- `UModal`
- `USlideover`
- `UDrawer`
- `UPopover`
- `UTooltip`
- `UDropdownMenu`
- `UContextMenu`

## Overlay Copy

Overlay titles should name the decision or object:

- `Review pattern`
- `Delete token`
- `Project settings`

Avoid generic titles such as `Are you sure?` when the action can be named directly.
