Navigation should make structure visible. It should help people understand where they are, what belongs together, and which path comes next.

::nuxt-ui-docs-link{links="UNavigationMenu|https://ui.nuxt.com/docs/components/navigation-menu; ULink|https://ui.nuxt.com/docs/components/link"}
::

Primary navigation shows peer destinations and direct paths. Active states should stay restrained, while labels should name the place or result clearly.

## Location And Views

::nuxt-ui-docs-link{links="UBreadcrumb|https://ui.nuxt.com/docs/components/breadcrumb; UTabs|https://ui.nuxt.com/docs/components/tabs"}
::

Breadcrumbs explain where the user is. Tabs switch local views inside one task or section. Neither should replace the page title.

::component-example{name="navigation-location-pattern"}
::

::nuxt-ui-docs-link{links="UStepper|https://ui.nuxt.com/docs/components/stepper; UAccordion|https://ui.nuxt.com/docs/components/accordion; UPagination|https://ui.nuxt.com/docs/components/pagination; UTree|https://ui.nuxt.com/docs/components/tree"}

## Navigation Rules

- Use tabs for sibling views, not unrelated destinations.
- Use breadcrumbs for location, not marketing hierarchy.
- Use accordions for optional detail, not essential content.
- Use steppers only when order matters.
- Keep navigation labels short and concrete.

## Navigation Labels

Good labels name the place or action directly:

- `Colors`
- `Typography`
- `Components`
- `Open docs`
- `Copy token`

Avoid decorative or abstract labels that make the user infer the destination.
