System helpers should stay quiet and precise. They support the product shell, accessibility, and recovery paths; they should not become decorative controls.

## Appearance And Theme Controls

::nuxt-ui-docs-link{links="UColorModeButton|https://ui.nuxt.com/docs/components/color-mode-button; UColorModeSwitch|https://ui.nuxt.com/docs/components/color-mode-switch; UColorModeSelect|https://ui.nuxt.com/docs/components/color-mode-select; UTheme|https://ui.nuxt.com/docs/components/theme"}
::

Color mode controls are global system affordances. Use a button for compact headers, a switch or select in settings, and theme scopes only for controlled previews.

::component-example{name="system-appearance-pattern"}
::

## Mode-Aware Assets

::nuxt-ui-docs-link{links="UColorModeAvatar|https://ui.nuxt.com/docs/components/color-mode-avatar; UColorModeImage|https://ui.nuxt.com/docs/components/color-mode-image"}
::

Mode-aware assets are for images that need contrast changes, such as wordmarks or diagrams with real light and inverse variants. The full symbol should keep its original colors.

::component-example{name="system-assets-pattern"}
::

## Access, Locale, And Recovery

::nuxt-ui-docs-link{links="ULocaleSelect|https://ui.nuxt.com/docs/components/locale-select; UAuthForm|https://ui.nuxt.com/docs/components/auth-form; UError|https://ui.nuxt.com/docs/components/error"}
::

Locale controls should stay close to language-sensitive content. Auth forms need one clear primary path, and recovery states should explain what happened and where to go next.
