System helpers should stay quiet and precise. They support the product shell, accessibility, and recovery paths; they should not become decorative controls.

::component-family-showcase{family="system-helpers"}
::

## Rules

- Use color-mode controls only where users reasonably expect them.
- Keep locale controls compact and close to language-sensitive content.
- Use auth forms as trusted product surfaces with plain labels and one clear primary action.
- Error pages should explain the state and offer a useful next step.
- Do not recolor the full happydesigns symbol for dark mode; use mode-aware assets only where contrast requires it.

## Covered Components

- `UColorModeButton`
- `UColorModeSwitch`
- `UColorModeSelect`
- `UColorModeAvatar`
- `UColorModeImage`
- `ULocaleSelect`
- `UAuthForm`
- `UError`
- `UTheme`

## Application Notes

System helpers are often global controls. Their styling should come from Nuxt UI defaults and semantic tokens so they stay consistent across docs, product shells, and embedded examples.
