Forms should feel calm and trustworthy. Labels stay plain, fields keep visible borders, and help text stays close to the decision it supports.

## Form Structure

::id-nuxt-ui-docs-link{links="UForm|https://ui.nuxt.com/docs/components/form; UFormField|https://ui.nuxt.com/docs/components/form-field; UFieldGroup|https://ui.nuxt.com/docs/components/field-group"}
::

Forms group related decisions without adding visual weight. Field labels, descriptions, errors, and adjacent actions should stay close to the control they explain.

## Selection Inputs

::id-nuxt-ui-docs-link{links="USelect|https://ui.nuxt.com/docs/components/select; USelectMenu|https://ui.nuxt.com/docs/components/select-menu; UInputMenu|https://ui.nuxt.com/docs/components/input-menu; UListbox|https://ui.nuxt.com/docs/components/listbox"}

Selection inputs solve different choice problems. Use selects for short known sets, searchable menus for larger sets, input menus for exact values with suggestions, and listboxes when options should stay visible.

## Validation Copy

Validation should explain the issue and the recovery path.

| Weak | Better |
| --- | --- |
| `Invalid input.` | `Use a valid project URL.` |
| `Something went wrong.` | `The file could not be uploaded. Check the format and try again.` |
| `Required.` | `Add a project name before saving.` |
