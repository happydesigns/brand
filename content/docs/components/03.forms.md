Forms should feel calm and trustworthy. Labels stay plain, fields keep visible borders, and help text stays close to the decision it supports.

::component-family-showcase{family="forms"}
::

## Rules

- Use direct labels such as `Email`, `Project name`, or `Status`.
- Keep help text short and practical.
- Use coral for focus and active details, not for every border.
- Use rose only for validation errors or destructive states.
- Keep technical values exact when they are tokens, versions, packages, or file names.

## Validation Copy

Validation should explain the issue and the recovery path.

| Weak | Better |
| --- | --- |
| `Invalid input.` | `Use a valid project URL.` |
| `Something went wrong.` | `The file could not be uploaded. Check the format and try again.` |
| `Required.` | `Add a project name before saving.` |
