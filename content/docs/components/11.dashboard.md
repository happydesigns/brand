Dashboard components carry dense product work. The happydesigns layer should make navigation, search, panels, and toolbars feel structured without turning the interface into a decorative brand moment.

## Application Shell

Dashboard components should be reviewed as one product shell. The sidebar, panel, navigation bar, toolbar, search, and density controls all shape the same surface. Test the shell in realistic density so repeated work stays readable without making every panel expressive.

Use `UDashboardSidebar` when the sidebar belongs to a `UDashboardGroup` and needs dashboard behavior such as collapse, resize, and panel coordination. Use `USidebar` for standalone navigation outside a dashboard shell.
