Dashboard components carry dense product work. The happydesigns layer should make navigation, search, panels, and toolbars feel structured without turning the interface into a decorative brand moment.

::component-family-showcase{family="dashboard"}
::

## Rules

- Keep dashboard navigation quiet, compact, and predictable.
- Use sidebars for persistent structure, not for brand decoration.
- Keep toolbar controls close to the data or panel they affect.
- Let resize handles and scroll areas stay subtle until the user interacts with them.
- Use strong actions sparingly inside dense product shells.

## Covered Components

- `UDashboardGroup`
- `UDashboardSidebar`
- `UDashboardPanel`
- `UDashboardNavbar`
- `UDashboardToolbar`
- `UDashboardResizeHandle`
- `UDashboardSearch`
- `UDashboardSearchButton`
- `UDashboardSidebarCollapse`
- `UDashboardSidebarToggle`
- `USidebar`
- `UScrollArea`

## Application Notes

Dashboard examples should be reviewed in real product density. The goal is not to make every panel expressive; the goal is to keep repeated work readable and stable.
