Actions should be clear before they feel branded. A user should understand the main action, secondary choices, and supporting metadata without reading around the interface.

::component-family-showcase{family="actions"}
::

## Rules

- Use graphite-filled primary buttons for strong actions.
- Use outlined or subtle secondary buttons for alternate paths.
- Keep coral available for focus, active state, and signature emphasis.
- Use icon-only buttons only when the symbol is familiar or has a tooltip.
- Avoid huge coral CTA surfaces by default.

## Covered Components

- `UButton`
- `UBadge`
- `UChip`
- `UKbd`
- `UTooltip`

## Copy

Use action labels that describe the result:

- `Save changes`
- `Copy token`
- `Open docs`
- `Start a project`

Avoid vague labels such as `Submit`, `Click here`, or `Learn more` when a more specific action exists.
