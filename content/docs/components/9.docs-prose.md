Docs surfaces should make decisions easy to read, cite, and reuse. Prose components carry explanation, examples, and implementation detail without becoming decorative.

## Callout

::note{title="Use callouts for context, not decoration"}
Callouts should clarify a decision, warning, or implementation note. They should not appear simply to add color to a page.
::

## Cards

::card-group
  :::card{title="Decision card" icon="i-lucide-list-checks"}
  Use cards when the whole block is a reusable piece of guidance.
  :::

  :::card{title="Implementation card" icon="i-lucide-code-2"}
  Use technical cards for exact commands, file names, and integration notes.
  :::
::

## Steps

::steps
### Set the base

Start with spacing, hierarchy, type, and color roles.

### Add recognition

Choose the logo form and voice that fit the context.

### Apply through components

Let Nuxt UI components carry the system consistently.
::

## Tabs

::tabs
  :::tabs-item{label="Usage" icon="i-lucide-book-open"}
  Use tabs for sibling views that belong to the same task.
  :::

  :::tabs-item{label="Code" icon="i-lucide-code-2"}
  Use code tabs when equivalent implementation paths need to sit together.
  :::
::
