Docs surfaces should make decisions easy to read, cite, and reuse. Prose components carry explanation, examples, and implementation detail without becoming decorative.

## Callouts For Decisions

::nuxt-ui-docs-link{links="Callout|https://ui.nuxt.com/docs/typography/callout"}
::

Use notes when a decision needs extra context, a warning, or a concrete implementation rule.

::callout{title="Use callouts for decisions" icon="i-lucide-info"}
Callouts should clarify a decision, warning, or implementation note. They should not be used as decoration between normal paragraphs.
::

## Cards For Reusable Guidance

::nuxt-ui-docs-link{links="Card|https://ui.nuxt.com/docs/typography/card; CardGroup|https://ui.nuxt.com/docs/typography/card-group"}
::

Use cards when a block of guidance should stand alone, be compared, or repeat across pages.

::card-group
  :::card{title="Decision card" icon="i-lucide-list-checks"}
  Use cards when the whole block is a reusable piece of guidance.
  :::

  :::card{title="Implementation card" icon="i-lucide-code-2"}
  Use technical cards for exact commands, file names, and integration notes.
  :::
::

## Ordered And Local Guidance

::nuxt-ui-docs-link{links="Steps|https://ui.nuxt.com/docs/typography/steps; Tabs|https://ui.nuxt.com/docs/typography/tabs"}
::

Use steps when order matters. Use tabs when several views belong to the same decision and should not stack vertically.

::steps
### Set the base

Start with spacing, hierarchy, type, and color roles.

### Add recognition

Choose the logo form and voice that fit the context.

### Apply through components

Let Nuxt UI components carry the system consistently.
::

::tabs
  :::tabs-item{label="Usage" icon="i-lucide-book-open"}
  Use tabs for sibling views that belong to the same task.
  :::

  :::tabs-item{label="Code" icon="i-lucide-code-2"}
  Use code tabs when equivalent implementation paths need to sit together.
  :::
::

## Code And Commands

::nuxt-ui-docs-link{links="CodeGroup|https://ui.nuxt.com/docs/typography/code-group; Code|https://ui.nuxt.com/docs/typography/code; Pre|https://ui.nuxt.com/docs/typography/pre"}
::

Use inline code for exact values. Use code blocks and code groups when the implementation itself is the example.

::code-group
```bash [pnpm]
pnpm add @happydesigns/brand
```

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['@happydesigns/brand']
})
```
::

```ts [app.config.ts]
export default defineAppConfig({
  ui: {
    button: {
      slots: {
        base: 'font-medium'
      }
    }
  }
})
```

## Tables And Fields

::nuxt-ui-docs-link{links="Table|https://ui.nuxt.com/docs/typography/table; Field|https://ui.nuxt.com/docs/typography/field; FieldGroup|https://ui.nuxt.com/docs/typography/field-group"}
::

Use tables for comparison and fields for exact options, tokens, or configuration values.

| Surface | Use |
| --- | --- |
| Callout | Important context, warning, or decision note |
| Code group | Install commands and implementation variants |
| Steps | Ordered guidance where order matters |

::field-group
  ::field{name="primary" type="string" default="#F28564"}
  Signature accent for focus, active states, and brand details.
  ::

  ::field{name="neutral" type="string" default="#242423"}
  Graphite carries strong actions and default text.
  ::
::

## Compact Inline Signals

::nuxt-ui-docs-link{links="Icon|https://ui.nuxt.com/docs/typography/icon; Badge|https://ui.nuxt.com/docs/typography/badge; Kbd|https://ui.nuxt.com/docs/typography/kbd"}
::

Use icons, badges, and keyboard hints only when they improve scanning or identify an exact action.

::card-group
  :::card{title="Icon" icon="i-lucide-sparkles"}
  Use an icon when it makes the category faster to recognize.
  :::

  :::card{title="Badge" icon="i-lucide-badge-check"}
  Use a badge for state, version, or compact metadata.
  :::
::

## Progressive Detail

::nuxt-ui-docs-link{links="Collapsible|https://ui.nuxt.com/docs/typography/collapsible; Accordion|https://ui.nuxt.com/docs/typography/accordion; CodeCollapse|https://ui.nuxt.com/docs/typography/code-collapse"}
::

Use disclosure when detail is useful but not required for the first read. Keep primary decisions visible.

::accordion
  :::accordion-item{label="When to collapse"}
  Collapse supporting implementation notes, not core brand rules.
  :::

  :::accordion-item{label="When to keep open"}
  Keep examples visible when they teach the visual behavior of a component.
  :::
::

::code-collapse
```ts [tokens.ts]
export const colors = {
  graphite: '#242423',
  warmWhite: '#FAF7F2',
  coral: '#F28564'
}
```
::
