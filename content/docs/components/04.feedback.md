Feedback should be calm, specific, and tied to the next useful action. Color should clarify state, not decorate the message.

::component-family-showcase{family="feedback"}
::

## Rules

- Use seafoam for success, availability, and constructive progress.
- Use butter for soft warnings.
- Use rose for errors, destructive states, and blocking validation.
- Use petrol or neutral treatments for informational guidance.
- Keep feedback copy short, specific, and recovery-oriented.

## Covered Components

- `UAlert`
- `UBanner`
- `UProgress`
- `USkeleton`
- `UBadge`
- `UToast`
- `UEmpty`
- `UIcon`

## State Copy

| State | Example |
| --- | --- |
| Success | `Changes saved.` |
| Progress | `Generating preview.` |
| Empty | `No projects yet. Start with the first project brief.` |
| Warning | `Check contrast before publishing.` |
| Error | `The token name is already used. Choose a unique name.` |
