Layout components define the room that interface components live in. They should make structure visible before any brand expression appears.

::component-family-showcase{family="layout-page"}
::

## Rules

- Use page primitives for repeated structure instead of rebuilding sections per page.
- Keep heroes focused on one promise and one clear path forward.
- Use sections, grids, and cards to group decisions by relationship.
- Keep CTAs practical; they should point to the next useful action.
- Use header, footer, container, and main surfaces as shared shell behavior, not page-level styling.

## Covered Components

- `UPageHero`
- `UPageSection`
- `UPageCTA`
- `UPageGrid`
- `UPageCard`
- `UPageFeature`
- `UPageLinks`
- `UContainer`
- `UMain`
- `UHeader`
- `UFooter`

## Shell Notes

`UHeader`, `UMain`, `UFooter`, and `UContainer` are best reviewed through the actual guide shell. They define the stable page frame and should stay calm, bordered, and predictable.

`UFooterColumns`, `UPageColumns`, `UPageList`, `UPageLogos`, and `UPageAnchors` still need dedicated brand review before they should be marked as fully showcased.
