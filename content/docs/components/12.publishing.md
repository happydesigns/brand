Publishing components are useful for release notes, announcements, package choices, and editorial surfaces. They should feel structured and calm, with expression reserved for the content itself.

::component-family-showcase{family="publishing"}
::

## Rules

- Use blog and changelog surfaces for dated product or brand decisions.
- Keep pricing and package comparisons practical; avoid sales-heavy styling.
- Use marquee motion only for controlled proof, partner, or token rows.
- Keep editorial surfaces readable before adding visual emphasis.
- Use the same border and surface rhythm as the rest of the guide.

## Covered Components

- `UBlogPosts`
- `UBlogPost`
- `UChangelogVersions`
- `UChangelogVersion`
- `UPricingPlans`
- `UPricingPlan`
- `UPricingTable`
- `UMarquee`

## Application Notes

These components support happydesigns product pages, package documentation, and release notes. They should not turn the brand guide into a marketing template.
