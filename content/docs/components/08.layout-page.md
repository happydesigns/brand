Layout components define the room that interface components live in. They should make structure visible before any brand expression appears.

## Page Openings And Bodies

::nuxt-ui-docs-link{links="UPageHero|https://ui.nuxt.com/docs/components/page-hero; UPageHeader|https://ui.nuxt.com/docs/components/page-header; UPageBody|https://ui.nuxt.com/docs/components/page-body"}
::

Heroes carry the page promise and one or two grounded next actions. Page headers and bodies keep documentation and product views predictable after that opening.

::component-example{name="page-opening-pattern"}
::

## Section And Grid Systems

::nuxt-ui-docs-link{links="UPageSection|https://ui.nuxt.com/docs/components/page-section; UPageGrid|https://ui.nuxt.com/docs/components/page-grid; UPageCard|https://ui.nuxt.com/docs/components/page-card; UPageFeature|https://ui.nuxt.com/docs/components/page-feature"}
::

Sections create repeatable rhythm. Grids, cards, and features organize related decisions without requiring custom layout code for every page.

::component-example{name="section-system-pattern"}
::

::nuxt-ui-docs-link{links="UPageLinks|https://ui.nuxt.com/docs/components/page-links; UPageColumns|https://ui.nuxt.com/docs/components/page-columns; UPageList|https://ui.nuxt.com/docs/components/page-list; UPageLogos|https://ui.nuxt.com/docs/components/page-logos; UPageAnchors|https://ui.nuxt.com/docs/components/page-anchors"}
