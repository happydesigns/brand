Chat and editor components should keep writing, review, and assistive workflows readable. The brand layer should support calm interaction, visible structure, and clear system feedback.

::component-family-showcase{family="chat-editor"}
::

## Rules

- Keep message streams readable before adding expressive details.
- Treat prompts as input surfaces, not as decorative cards.
- Tool and reasoning displays should explain system behavior without overwhelming the user.
- Editor controls should support writing and review without dominating the content.
- Use client-only previews where components depend on browser-only behavior.

## Application Notes

These examples are visual coverage for the brand layer. The guide should not duplicate implementation details for AI transport, persistence, or editor schemas.
