Data and content components should make information easier to compare, scan, and reuse. The brand layer should support structure instead of making every surface expressive.

::component-family-showcase{family="data-content"}
::

## Rules

- Use tables when comparison matters.
- Use cards for repeated items, previews, and genuinely contained tools.
- Use separators and metadata to clarify structure.
- Use mono only for exact technical content.
- Do not nest cards inside cards.

## Covered Components

- `UTable`
- `UCard`
- `UPageCard`
- `UAvatar`
- `UAvatarGroup`
- `UUser`
- `USeparator`
- `UCollapsible`
- `UCarousel`
- `UTimeline`
- Prose tables
- Prose code blocks
- Prose callouts

## Content Surfaces

Technical surfaces include code blocks, token tables, package references, changelog notes, and implementation examples.

They should feel precise and quiet: mono for exact values, visible borders for structure, and restrained color for emphasis.
